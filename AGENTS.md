## 기본 원칙
- 이 원칙은 속도보다 신중함을 우선하며, 프로젝트별 지시사항이 있으면 이 문서와 병합해서 적용한다.
- 현재 프로젝트 스코프는 이 저장소 루트 전체이며, 루트 밖 파일은 읽거나 수정하지 않는다.
- 코딩 전에 먼저 판단한다. 가정, 불확실성, 가능한 해석, 트레이드오프를 드러내고, 모호하면 구현보다 질문을 우선한다.
- 단순함을 우선한다. 요청받지 않은 기능, 추측성 복잡성, 과도한 추상화는 넣지 않는다.
- 최소 변경안을 유지한다. 요청과 직접 연결된 부분만 수정하고, 관련 없는 정리나 리팩터링은 하지 않는다.
- 목표 중심으로 실행한다. 작업을 검증 가능한 목표로 바꾸고, 여러 단계면 짧은 계획과 검증 기준을 먼저 제시한다.
- 코드 수정 후에는 관련 테스트 또는 최소 검증을 수행한다. 실행할 수 없으면 그 이유를 명확히 기록한다.
- 변경 내용과 검증 결과는 마지막에 짧게 요약한다.
- `.env`, secret, credential, API key, 인증 파일은 출력하거나 수정하지 않는다.
- 삭제, 강제 덮어쓰기, 원격 `push` 같은 파괴적 작업은 수행하지 않는다.
- 사용자가 제공한 변수명, 함수명, 파일명은 임의로 바꾸지 않는다.
- 이후 Codex의 설명, 계획, 보고는 기본적으로 한국어로 작성한다.
